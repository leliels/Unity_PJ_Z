## ⚠️ 关于「会议内容」目录

`设计文档/会议内容/` 下的文件**仅为会议原始记录和梳理笔记**，不属于正式设计文档。

- 在进行任何参照文档的开发工作时，**不需要关注**会议内容目录下的文件
- 正式的设计规则和数值配置已同步到上方 01~07 的对应文档中
- 会议内容仅供回溯讨论过程时参考

## 🔄 临时迭代流程（SOP）

当团队成员在试玩当前版本的过程中产生了新需求、优化想法或发现问题时，按以下流程处理：

### 流程步骤

```
1. 提出需求/问题
   ↓
2. 更新设计文档（02~05 中的对应章节）
   ↓
3. 更新开发计划（06-开发计划.md 中当前里程碑的任务列表）
   ↓
4. 实施开发
   ↓
5. 验证效果 → 如有问题回到 步骤1
```

### 规则

- **先文档后代码**：任何功能变更都必须先写入设计文档，再进行代码实施
- **当前里程碑优先**：临时需求优先归入当前正在进行的里程碑；如果工作量较大则标注为下一里程碑
- **保持文档同步**：代码实施完成后，更新开发计划中的 checkbox 状态和开发日志
- **会议内容不算正式文档**：如果需求来自会议讨论，结论必须同步到正式文档（01~07）中才生效
