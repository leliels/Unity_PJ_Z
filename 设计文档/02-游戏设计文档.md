# 02 - 游戏设计文档

## 1. 游戏概述

一款类似《快乐爱消除》的 2D 休闲方块消除游戏。玩家在固定大小的棋盘上放置不同形状的方块，填满整行或整列即可消除得分。通过关卡模式一关一关推进，每关有特定的目标和步数限制。

## 2. 核心玩法规则

### 2.1 棋盘

- 固定大小的方格棋盘（**待确认：8×8 还是 10×10？建议先做 8×8**）
- 每个格子有两种状态：空 / 已占用
- 棋盘初始状态为全空（或根据关卡配置有预置方块）

### 2.2 方块

- 屏幕下方的**候选区**同时显示 **3 个**方块组合供玩家选择
- 方块组合由若干小格子组成，形状各异（类似俄罗斯方块的形状）
- 玩家用手指/鼠标**拖拽**方块到棋盘上的空位放置
- 方块只能放在完全空的位置上，不能重叠
- 放置后不能移动、不能旋转（**待确认：是否允许旋转？**）
- 3 个候选方块用完后，自动刷新新的 3 个

### 2.3 方块形状表（初步设定）

| 竖二 | ■<br>■ | 2 |
| 横三 | ■■■ | 3 |

### 2.4 消除机制

- **行消除**：某一行的所有格子都被填满 → 该行所有方块被消除
- **列消除**：某一列的所有格子都被填满 → 该列所有方块被消除
- **同时消除**：如果一次放置同时填满了多行/多列，全部同时消除（连消奖励）
- 消除后，上方的方块**不会**下落（区别于传统三消游戏）

#### 2.4.1 消除颜色覆盖提示（2026-04-24 新增）

当方块放到棋盘上触发消除时，**被消除的那一整排会先变成手上拿着的方块的颜色**，形成一个短暂的颜色覆盖效果，然后再执行消除动画。

- **设计意图**：让玩家清楚看到"我的操作影响了哪些格子"，是一种**视觉提示**
- **表现流程**：放置方块 → 满行/列检测 → 颜色覆盖（约 0.2~0.3 秒） → 消除动画 → 格子清空
- **优先级**：高（参考竞品核心体验，对理解反馈非常重要）

> 来源：2026-04-24 会议，参考竞品"快来消除"的设计。

### 2.5 计分规则（2026-04-23 确认版）

#### 2.5.1 放置方块得分

- 每放置一个方块，按方块占格数加分
- **1 格 = 1 分**（例：L 形占 3 格 = 3 分，田字形占 4 格 = 4 分）

#### 2.5.2 消除得分（幂次增长）

棋盘为 8×8，消一排 = 消除 8 个格子。消除基数 = 8 × 2 = **16 分**。

**公式：消除分 = 16^N**（N = 消除的行/列数）

| 消除行/列数 | 计算方式 | 得分 |
|---|---|---|
| 1 排 | 16¹ | 16 |
| 2 排 | 16² | 256 |
| 3 排 | 16³ | 4,096 |
| 4 排 | 16⁴ | 65,536 |

> 核心设计意图：通过幂次增长拉开多排消除的爽感差距，激励玩家追求一次消除更多行/列。

#### 2.5.3 Combo 连击系统

- **Combo 定义**：连续放置方块且每次都触发消除，即构成 Combo 连击
- **Combo 加成方式**：在**本次消除得分**的基础上，乘以 Combo 系数（不是乘总分）
- **Combo 系数规则**：每多一次 Combo，系数增加 0.2

| Combo 次数 | 系数 | 说明 |
|---|---|---|
| 第 1 次消除 | ×1.0 | 不算 Combo |
| Combo ×1 | ×1.2 | 第 2 次连续消除 |
| Combo ×2 | ×1.4 | 第 3 次连续消除 |
| Combo ×3 | ×1.6 | 第 4 次连续消除 |
| Combo ×N | ×(1 + N×0.2) | 以此类推 |

- Combo 中断条件：放置方块后**未触发**任何消除，Combo 重置为 0
- 每次 Combo 的消除行数不同 → 基数不同，但 Combo 系数统一按 +0.2 递增

#### 2.5.4 单次得分计算流程

```
1. 放置方块 → +方块占格数 分
2. 检测消除 → 计算消除分 = 16^(消除行列数)
3. 应用 Combo → 消除分 × Combo 系数
4. 本步总分 = 放置分 + 消除分(含 Combo)
5. 累加到总分
```

> 数值公式为初版设计，后续根据实际手感可微调。
> 单次积得分跟新
1. 放置方块 → +方块占格数分
2. 检测消除 → 计算消除分 =  20^(消除行列数)
3. 应用 Combo → {消除分（20）+（19 × Combo 系数）` 
4. 本步总分 = 放置分 + 消除分(含 Combo)
5. 累加到总分 

6. A+(B*排分)+排数*[20*Combo数*（1+排数）]  其中：A=当前分数 B=当前放下方块分数 排分：消除一排=1排分 两排=3排分 三排=5排分 四排=7排分 五排=9排分  每次消除1排就算一次combo，同时消除2排就是2次连击
#### 2.5.5 得分表现系统

消除发生后，需要给玩家**逐步展示**本次获得的所有分数，而不是直接加到总分里。表现流程如下：

```
消除发生
  ↓
① 显示放置分飘字（如 "+4"），短暂停留后消失
  ↓
② 逐行显示消除分飘字（每消除一行/列 → 显示一条，如 "+16"、"+256"）
  ↓
③ 如果有 Combo → 显示 Combo 加成飘字（如 "Combo ×1 → ×1.2"）
  ↓
④ 所有飘字展示完毕后 → 汇总加入总分 → 总分数字有跳动效果
  ↓
⑤ 飘字淡出消失
```

**飘字规格：**
- 位置：在棋盘上方区域，从消除位置向上飘动
- 样式：大号数字 + 颜色区分（放置分白色、消除分金色、Combo 分彩色）
- 动画：数字从小到大弹出 → 短暂停留 → 上飘淡出
- 节奏：每条飘字之间间隔 0.2~0.3 秒，形成逐个"打"到屏幕上的节奏感

**总分跳动效果：**
- 所有飘字播完后，总分数字从旧值跳动到新值（类似老虎机滚动效果）
- 后续可进一步打磨为类似《小丑牌 Balatro》的逐步计算动画

> 注：动画细节为初版方案，后续根据手感打磨。M2 阶段先实现基础飘字功能，动画打磨放到 M4/M5。

## 3. 关卡系统

### 3.1 关卡基本要素

每个关卡包含以下配置参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 关卡编号 | 唯一标识 | Level_001 |
| 目标分数 | 达到此分数即过关 | 500 |
| 步数限制 | 可放置方块的总次数 | 20 |
| 棋盘大小 | 棋盘的行×列 | 8×8 |
| 预置方块 | 开局时已有的方块 | 待补充 |
| 可用方块池 | 本关可出现的方块形状 | 待补充 |
| 星级评价 | 1星/2星/3星的分数线 | 待补充 |

### 3.2 关卡流程

```
进入关卡 → 显示关卡目标 → 开始游戏
  ↓
每一步：选择候选方块 → 拖拽放置 → 检查消除 → 更新分数 → 剩余步数-1
  ↓
判定结果：
  ├─ 分数 ≥ 目标分数 → 过关！显示星级评价
  └─ 步数用尽且分数不足 → 失败，可选重试或返回
```

### 3.3 关卡进度存档

- 记录玩家已通过的最高关卡
- 记录每关的最高分和星级
- **存储方式**：使用 Unity 的 `PlayerPrefs`（简单方案，够用即可）
