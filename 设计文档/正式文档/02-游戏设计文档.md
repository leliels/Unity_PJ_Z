# 02 - 游戏设计文档

## 1. 游戏概述

一款类似《快乐爱消除》的 2D 休闲方块消除游戏。玩家在固定大小的棋盘上放置不同形状的方块，填满整行或整列即可消除得分。通过关卡模式一关一关推进，每关有特定的目标和步数限制。

## 2. 核心玩法规则

### 2.1 棋盘

- 固定大小的方格棋盘（**待确认：8×8 还是 10×10？建议先做 8×8**）
- 每个格子有两种状态：空 / 已占用
- 棋盘初始状态为全空（或根据关卡配置有预置方块）

### 2.2 方块

- 屏幕下方的**候选区**同时显示 **3 个**方块组合供玩家选择
- 方块组合由若干小格子组成，形状各异（类似俄罗斯方块的形状）
- 玩家用手指/鼠标**拖拽**方块到棋盘上的空位放置
- 方块只能放在完全空的位置上，不能重叠
- 放置后不能移动、不能旋转（**待确认：是否允许旋转？**）
- 3 个候选方块用完后，自动刷新新的 3 个

### 2.3 方块形状表（配置化）

方块形状由 Unity 编辑器工具维护，不再以设计文档中的固定表格作为最终来源。运行时 `BlockSpawner` 会优先读取当前配置的 `BlockShapeDatabase`，从其中启用且权重大于 0 的形状里按权重随机出块；如果数据库缺失或无有效形状，则回退到代码内置形状表，保证游戏不崩溃。

当前工具支持：

- 图形化点格子配置方块形状；
- 单个形状启用/禁用；
- 每个形状独立配置出现权重；
- 一键生成 0/90/180/270 四个方向变体；
- 多份方块数据库，用于不同模式、难度或测试。

具体操作方式见 [操作指南/方块形状配置工具使用指南](操作指南/方块形状配置工具使用指南.md)。

### 2.4 消除机制

- **行消除**：某一行的所有格子都被填满 → 该行所有方块被消除
- **列消除**：某一列的所有格子都被填满 → 该列所有方块被消除
- **同时消除**：如果一次放置同时填满了多行/多列，全部同时消除（连消奖励）
- 消除后，上方的方块**不会**下落（区别于传统三消游戏）

#### 2.4.1 消除颜色覆盖提示（2026-04-24 新增）

当方块放到棋盘上触发消除时，**被消除的那一整排会先变成手上拿着的方块的颜色**，形成一个短暂的颜色覆盖效果，然后再执行消除动画。

- **设计意图**：让玩家清楚看到"我的操作影响了哪些格子"，是一种**视觉提示**
- **表现流程**：放置方块 → 满行/列检测 → 颜色覆盖（约 0.2~0.3 秒） → 消除动画 → 格子清空
- **优先级**：高（参考竞品核心体验，对理解反馈非常重要）

> 来源：2026-04-24 会议，参考竞品"快来消除"的设计。

### 2.5 计分规则（2026-04-29 配置化确认版）

> 本节为当前正式计分框架。旧版 `16^N` 幂次计分和 `Combo 系数 +0.2` 规则不再作为当前实现依据，仅保留在历史记录中。

计分由三部分组成：

1. **放置方块得分**
2. **消除基础加分**
3. **Combo 连击加成**

一次操作的总分更新逻辑为：

```text
总分 += 放置分
如果本次触发消除：
    总分 += 消除基础加分 + Combo加成
```

#### 2.5.1 放置方块得分

每次成功放置方块，按照方块实际占用格子数立即加分。

```text
放置分 B = 方块占格数 × 每格放置分
```

默认配置：

| 配置项 | 默认值 | 说明 |
|---|---:|---|
| 每格放置分 | 1 | 每占用 1 个棋盘格，放置成功后立即获得 1 分 |

示例：

- 3 格方块：`3 × 1 = 3 分`
- 4 格方块：`4 × 1 = 4 分`

#### 2.5.2 消除排数

一次放置后，可能同时消除多条行或列。

```text
消除排数 L = 本次同时被消除的行数 + 列数
```

示例：

- 消除 1 行：`L = 1`
- 同时消除 1 行 + 1 列：`L = 2`
- 同时消除 2 行 + 1 列：`L = 3`

#### 2.5.3 排分

排分用于放大多排消除的收益。

```text
排分 R = 根据本次消除排数 L 查表获得
```

默认排分表：

| 消除排数 L | 排分 R |
|---:|---:|
| 1 | 1 |
| 2 | 3 |
| 3 | 5 |
| 4 | 7 |
| 5 | 9 |

如果消除排数超过排分表长度，则从最后一档开始，按“超出排分递增值”继续递增。默认超出排分递增值为 `2`。

示例：默认表只有 5 档时，`6 排 = 11`，`7 排 = 13`。

#### 2.5.4 消除基础加分

本次触发消除时，消除基础加分与“本次放置分”和“排分”相关。

```text
消除基础加分 = 放置分 B × 排分 R
```

示例：

- 本次放置 4 格方块，消除 1 排：`4 × 1 = 4`
- 本次放置 4 格方块，消除 2 排：`4 × 3 = 12`
- 本次放置 5 格方块，消除 3 排：`5 × 5 = 25`

注意：放置分 B 已经在放置成功时加过一次；这里再次使用 B，是把它作为本次消除收益的计算因子，不是重复描述同一个加分步骤。

#### 2.5.5 Combo 连击加成

Combo 用于奖励连续发生的消除行为。为避免混淆，先把三个概念分开：

1. **Combo 数 C**
   - 表示本轮连续消除已经积累的 Combo 强度。
   - 默认从 `0` 开始。
   - 每次消除后，根据本次消除排数增加：`C += 消除排数 L × 每排 Combo 增长值`。
   - 默认配置下，消除 1 排增加 1 点 Combo；消除 2 排增加 2 点 Combo。
   - Combo 数默认不会逐步减少；只会在 Combo 中断、奖励机会耗尽或规则要求重置时归零。

2. **Combo 加成系数 K**
   - 来自当前新公式中的 `20`。
   - 它不是 Combo 数，也不是奖励机会数。
   - 它只控制 Combo 额外分的整体强度：数值越大，Combo 给的额外分越高。

3. **Combo 奖励机会数 W**
   - 表示本轮 Combo 中，还剩多少次“可以获得 Combo 加成”的机会。
   - 默认第一次消除开启一轮 Combo，并设置 `W = 每轮 Combo 奖励机会数`。
   - 每次真正触发 Combo 加成时，扣除 `每次触发消耗机会数`。
   - 默认情况下，W 不会因为消除而自动加回去。
   - 当 W 归零，本轮 Combo 结束，Combo 数 C 清零；下一次消除会重新开启新一轮。
   - 如果希望“越连越久”，可调高“每次消除补充机会数”。默认补充为 `0`。

Combo 加成公式：

```text
Combo加成 = 消除排数 L × Combo加成系数 K × 当前Combo数 C × (1 + 消除排数 L)
```

默认 Combo 流程：

```text
1. 首次消除：C = 0，W = 每轮Combo奖励机会数，本次默认不产生Combo加成
2. 后续消除：如果 W > 0，则用“本次消除前的 C”计算 Combo加成
3. 本次消除结算后：C += L × 每排Combo增长值
4. 如果本次触发了 Combo加成：W -= 每次触发消耗机会数
5. 如果配置了“每次消除补充机会数”：W += 每次消除补充机会数，但不超过每轮上限
6. 如果 W <= 0：本轮 Combo 结束，C 重置为 0，等待下一次消除重新开启
7. 如果本次没有消除：默认重置 C 和 W；是否重置由配置开关决定
```

默认配置：

| 配置项 | 默认值 | 说明 |
|---|---:|---|
| 启用 Combo | 开启 | 是否启用 Combo 连击加成 |
| Combo 加成系数 K | 20 | 控制 Combo 额外分整体强度 |
| 每轮 Combo 奖励机会数 W | 3 | 一轮 Combo 中最多允许几次后续消除获得 Combo 加成 |
| 每次触发消耗机会数 | 1 | 每次实际获得 Combo 加成后，从 W 中扣除多少次 |
| 每次消除补充机会数 | 0 | 每次消除后给 W 补回多少；默认不补充 |
| 首次消除是否计算 Combo | 关闭 | 关闭时，首次消除只开启 Combo 轮次，不产生 Combo 加成 |
| 每排 Combo 增长值 | 1 | 每消除 1 排增加多少 Combo 数 |
| 未消除是否重置 Combo | 开启 | 放置方块但没有消除时，是否立即清空 C 和 W；默认重置 |

#### 2.5.6 单次得分计算流程

```text
1. 玩家放置方块
2. 计算放置分 B，并立即加入总分
3. 检测是否产生消除
4. 如果没有消除：
   - 本次计分结束
   - 默认重置 Combo 数 C 和奖励机会数 W
   - 是否重置可由“未消除是否重置 Combo”配置开关控制
5. 如果产生消除：
   - 计算消除排数 L
   - 根据 L 查表得到排分 R
   - 计算消除基础加分 = B × R
   - 根据当前 C、W 和配置决定是否计算 Combo 加成
   - 将 消除基础加分 + Combo加成 加入总分
   - 按配置更新 C 和 W
```

| 排分表 | 1,3,5,7,9 | 根据本次消除排数查表得到排分 |
| 超出排分递增值 | 2 | 当消除排数超过排分表长度时，每多 1 排额外增加多少排分 |

| 每轮 Combo 奖励机会数 | 3 | 一轮 Combo 开启后，最多允许多少次后续消除获得 Combo 加成 |

#### 2.5.8 得分表现系统

消除发生后，需要给玩家逐步展示本次获得的主要分数项。表现流程如下：

```text
消除发生
  ↓
① 显示本次放置分飘字（如 "+4"）
  ↓
② 显示消除基础加分飘字（如 "+12"）
  ↓
③ 如果有 Combo 加成 → 显示 Combo 加成飘字（如 "Combo ×2 +240"）
  ↓
④ 所有飘字展示完毕后 → 总分数字有跳动效果
  ↓
⑤ 飘字淡出消失
```

**飘字规格：**
- 位置：在棋盘上方区域，从消除位置向上飘动
- 样式：大号数字 + 颜色区分（放置分白色、消除基础加分金色、Combo 加成彩色）
- 动画：数字从小到大弹出 → 短暂停留 → 上飘淡出
- 节奏：每条飘字之间间隔 0.2~0.3 秒，形成逐个“打”到屏幕上的节奏感

**总分跳动效果：**
- 所有飘字播完后，总分数字播放跳动效果
- 后续可进一步打磨为类似《小丑牌 Balatro》的逐步计算动画

> 注：动画细节为初版方案，后续根据手感打磨。M2 阶段先实现基础飘字功能，动画打磨放到 M4/M5。

## 3. 关卡系统

### 3.1 关卡基本要素

每个关卡包含以下配置参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 关卡编号 | 唯一标识 | Level_001 |
| 目标分数 | 达到此分数即过关 | 500 |
| 步数限制 | 可放置方块的总次数 | 20 |
| 棋盘大小 | 棋盘的行×列 | 8×8 |
| 预置方块 | 开局时已有的方块 | 待补充 |
| 可用方块池 | 本关可出现的方块形状 | 待补充 |
| 星级评价 | 1星/2星/3星的分数线 | 待补充 |

### 3.2 关卡流程

```
进入关卡 → 显示关卡目标 → 开始游戏
  ↓
每一步：选择候选方块 → 拖拽放置 → 检查消除 → 更新分数 → 剩余步数-1
  ↓
判定结果：
  ├─ 分数 ≥ 目标分数 → 过关！显示星级评价
  └─ 步数用尽且分数不足 → 失败，可选重试或返回
```

### 3.3 关卡进度存档

- 记录玩家已通过的最高关卡
- 记录每关的最高分和星级
- **存储方式**：使用 Unity 的 `PlayerPrefs`（简单方案，够用即可）
