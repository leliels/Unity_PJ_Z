# 02 - 游戏设计文档

## 1. 游戏概述

一款类似《快乐爱消除》的 2D 休闲方块消除游戏。玩家在固定大小的棋盘上放置不同形状的方块，填满整行或整列即可消除得分。通过关卡模式一关一关推进，每关有特定的目标和步数限制。

## 2. 核心玩法规则

### 2.1 棋盘

- 固定大小的方格棋盘（**待确认：8×8 还是 10×10？建议先做 8×8**）
- 每个格子有两种状态：空 / 已占用
- 棋盘初始状态为全空（或根据关卡配置有预置方块）

### 2.2 方块

- 屏幕下方的**候选区**同时显示 **3 个**方块组合供玩家选择
- 方块组合由若干小格子组成，形状各异（类似俄罗斯方块的形状）
- 玩家用手指/鼠标**拖拽**方块到棋盘上的空位放置
- 方块只能放在完全空的位置上，不能重叠
- 放置后不能移动、不能旋转（**待确认：是否允许旋转？**）
- 3 个候选方块用完后，自动刷新新的 3 个

### 2.3 方块形状表（配置化）

方块形状由 Unity 编辑器工具维护，不再以设计文档中的固定表格作为最终来源。运行时 `BlockSpawner` 会优先读取当前配置的 `BlockShapeDatabase`，从其中启用且权重大于 0 的形状里按权重随机出块；如果数据库缺失或无有效形状，则回退到代码内置形状表，保证游戏不崩溃。

当前工具支持：

- 图形化点格子配置方块形状；
- 单个形状启用/禁用；
- 每个形状独立配置出现权重；
- 一键生成 0/90/180/270 四个方向变体；
- 多份方块数据库，用于不同模式、难度或测试。

具体操作方式见 [操作指南/方块形状配置工具使用指南](操作指南/方块形状配置工具使用指南.md)。

### 2.4 消除机制

- **行消除**：某一行的所有格子都被填满 → 该行所有方块被消除
- **列消除**：某一列的所有格子都被填满 → 该列所有方块被消除
- **同时消除**：如果一次放置同时填满了多行/多列，全部同时消除（连消奖励）
- 消除后，上方的方块**不会**下落（区别于传统三消游戏）

#### 2.4.1 消除颜色覆盖提示（2026-04-24 新增）

当方块放到棋盘上触发消除时，**被消除的那一整排会先变成手上拿着的方块的颜色**，形成一个短暂的颜色覆盖效果，然后再执行消除动画。

- **设计意图**：让玩家清楚看到"我的操作影响了哪些格子"，是一种**视觉提示**
- **表现流程**：放置方块 → 满行/列检测 → 颜色覆盖（约 0.2~0.3 秒） → 消除动画 → 格子清空
- **优先级**：高（参考竞品核心体验，对理解反馈非常重要）

> 来源：2026-04-24 会议，参考竞品"快来消除"的设计。

### 2.5 计分规则（2026-04-30 正式版）

> 本节为当前正式计分框架，来源：2026-04-30 晚上在线会议。项目代码与默认配置均以本节为准。

#### 2.5.1 核心原则

1. **普通放置不直接加分**：玩家成功放置方块但没有触发消除时，当前分不变。
2. **分数只在触发消除时结算**：一次消除的得分由“格子得分项”和“消除/Combo 得分项”组成。
3. **Combo 数最小为 1**：Combo 被重置时回到 `1`，不再归零。
4. **Combo CD 用来判定续 Combo**：未消除会消耗 Combo CD；消除会重置 Combo CD。

一次触发消除时的总分更新逻辑为：

```text
当前分 A = 当前分 A + 格子得分项 + 消除/Combo得分项
```

#### 2.5.2 参数定义

| 符号 | 名称 | 默认值 | 是否配置 | 说明 |
|---|---|---:|---|---|
| A | 当前分 | 0 | 否 | 玩家当前总分，由计分流程实时更新 |
| B | 消除基本分 | 20 | 是 | 消除一排时使用的基础底分；支持配置为随机区间，例如 `19~21`；默认固定为 `20` |
| C | 格子基础分 | 由方块决定 | 否 | 本次放置方块由几个小格组成就是几，例如 4 格方块则 `C = 4` |
| D | 格子基础分倍率 | 1 | 是 | 用来放大或缩小 `C` 的倍率 |
| L | 消除排数 | 本次计算 | 否 | 本次同时消除的行数 + 列数 |
| R | 排分倍率 | 查表 | 是 | 根据 `L` 从排分倍率表中查得，用于奖励多排消除 |
| M | Combo 数 | 1 | 状态值 | 游戏过程中变化，最小为 `1`；重置时回到 `1` |
| N | Combo 增长值 | 1 | 是 | 每消除 1 排，增加多少 Combo 数 |
| CCD | Combo CD 值 | 3 | 是 | 消除后还有几次“未消除放置”的机会可以续上 Combo |

#### 2.5.3 排分倍率 R

一次放置后，可能同时消除多条行或列。

```text
消除排数 L = 本次同时被消除的行数 + 列数
排分倍率 R = 根据 L 查表获得
```

默认排分倍率表：

| 消除排数 L | 排分倍率 R |
|---:|---:|
| 1 | 1 |
| 2 | 3 |
| 3 | 5 |
| 4 | 7 |
| 5 | 9 |

如果未来出现超过配置表长度的消除排数，运行时应使用最后一档作为兜底值，避免计分异常；正式数值可通过扩展排分倍率表处理。

#### 2.5.4 Combo 状态规则

Combo 由 `Combo 数 M` 和 `Combo CD 值 CCD` 共同决定。

**初始状态：**

```text
M = 1
CCD = Combo CD 默认值
```

**没有触发消除时：**

```text
CCD = CCD - 1
如果 CCD <= 0：
    CCD = 0
    M = 1
当前分 A 不变
```

说明：

- 未消除不会扣分，也不会加分。
- 只要 `CCD` 还没有降到 `0`，`M` 保持不变。
- 一旦 `CCD` 降到 `0`，本轮 Combo 断开，`M` 重置为 `1`。

**触发消除时：**

```text
M = M + L × N
CCD = Combo CD 默认值
```

说明：

- Combo 数先增长，再参与本次得分公式。
- 消除后，Combo CD 重置为配置的默认值，例如默认 `3`。

#### 2.5.5 单次得分公式

当本次放置没有触发消除：

```text
A 不变
```

当本次放置触发消除：

```text
M = M + L × N
CCD = Combo CD 默认值
本次得分 = C × D × R + B × M × L × (L + 1)
A = A + 本次得分
```

其中：

```text
格子得分项 = C × D × R
消除/Combo得分项 = B × M × L × (L + 1)
```

`B` 在每次触发消除时取值一次：如果配置为固定值，例如 `20~20`，则本次 `B = 20`；如果配置为随机区间，例如 `19~21`，则本次在该区间内随机取值。

示例（默认配置：`B = 20`，`D = 1`，`N = 1`，`CCD = 3`）：

| 场景 | 计算过程 | 本次得分 |
|---|---|---:|
| 4 格方块消除 1 排，当前 `M = 1` | 先更新 `M = 1 + 1 × 1 = 2`；`R = 1`；得分 `= 4 × 1 × 1 + 20 × 2 × 1 × 2` | 84 |
| 5 格方块消除 2 排，当前 `M = 2` | 先更新 `M = 2 + 2 × 1 = 4`；`R = 3`；得分 `= 5 × 1 × 3 + 20 × 4 × 2 × 3` | 495 |
| 未触发消除，当前 `CCD = 3` | `CCD = 2`，`M` 不变，`A` 不变 | 0 |

#### 2.5.6 单次计分流程

```text
1. 玩家放置方块
2. 获取本次方块占格数 C
3. 检测是否产生消除
4. 如果没有消除：
   - CCD = CCD - 1
   - 如果 CCD <= 0，则 CCD = 0，M = 1
   - 当前分 A 不变
   - 本次计分结束
5. 如果产生消除：
   - 计算消除排数 L
   - 根据 L 查表得到排分倍率 R
   - 本次取值消除基本分 B
   - 先更新 Combo 数：M = M + L × N
   - 重置 Combo CD：CCD = Combo CD 默认值
   - 计算本次得分：C × D × R + B × M × L × (L + 1)
   - 将本次得分加入当前分 A
```

#### 2.5.7 计分配置化要求

计分框架固定为本节描述的结构，但关键数值需要从配置资源中读取，避免写死在脚本里。

| Inspector 中文名 | 默认值 | 说明 |
|---|---:|---|
| 消除基本分最小值 | 20 | `B` 的随机区间下限；默认与最大值相同，表示固定 20 分 |
| 消除基本分最大值 | 20 | `B` 的随机区间上限；如需轻微随机，可配置为 21，并将最小值配置为 19 |
| 格子基础分倍率 | 1 | `D`，用于放大或缩小本次方块占格数 `C` 的贡献 |
| 排分倍率表 | 1,3,5,7,9 | 根据本次消除排数 `L` 查表得到 `R` |
| Combo 初始值/最小值 | 1 | `M` 的初始值和重置值；固定为 1 |
| Combo 增长值 | 1 | `N`，每消除 1 排增加多少 Combo 数 |
| Combo CD 默认值 | 3 | 消除后 `CCD` 重置到该值；未消除时逐次递减 |
| 分数上限 | 2,147,483,647 | 防止公式配置过大导致分数溢出 |

所有开放给策划调整的字段，在 Unity Inspector 中应显示中文名，并提供中文悬浮说明。

具体使用哪个计分配置，由 Boot 场景中 `SceneBootstrap` 的 **本场景使用的计分配置** 字段指定。默认已绑定 `Assets/Resources/Configs/ScoreConfig.asset`。如果该字段为空，程序会兜底加载 `Resources/Configs/ScoreConfig`；如果未来有不同模式，可以创建多个 `ScoreConfig` 资源，并在进入对应模式/场景时指定不同配置。

#### 2.5.8 得分表现系统

新规则下，只有触发消除时才展示得分飘字；未触发消除时不展示得分飘字。消除发生后，需要给玩家逐步展示本次获得的主要分数项。

表现流程如下：

```text
消除发生
  ↓
① 显示格子得分项飘字（如 "+4"）
  ↓
② 显示消除/Combo得分项飘字（如 "Combo ×2 +80"）
  ↓
③ 所有飘字展示完毕后 → 总分数字有跳动效果
  ↓
④ 飘字淡出消失
```

**飘字规格：**
- 位置：在棋盘上方区域，从消除位置向上飘动
- 样式：大号数字 + 颜色区分（格子得分项白色、消除/Combo 得分项彩色）
- 动画：数字从小到大弹出 → 短暂停留 → 上飘淡出
- 节奏：每条飘字之间间隔 0.2~0.3 秒，形成逐个“打”到屏幕上的节奏感

**总分跳动效果：**
- 所有飘字播完后，总分数字播放跳动效果
- 后续可进一步打磨为类似《小丑牌 Balatro》的逐步计算动画

> 注：动画细节为初版方案，后续根据手感打磨。M2 阶段先实现基础飘字功能，动画打磨放到 M4/M5。



## 3. 关卡系统

### 3.1 关卡基本要素

每个关卡包含以下配置参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 关卡编号 | 唯一标识 | Level_001 |
| 目标分数 | 达到此分数即过关 | 500 |
| 步数限制 | 可放置方块的总次数 | 20 |
| 棋盘大小 | 棋盘的行×列 | 8×8 |
| 预置方块 | 开局时已有的方块 | 待补充 |
| 可用方块池 | 本关可出现的方块形状 | 待补充 |
| 星级评价 | 1星/2星/3星的分数线 | 待补充 |

### 3.2 关卡流程

```
进入关卡 → 显示关卡目标 → 开始游戏
  ↓
每一步：选择候选方块 → 拖拽放置 → 检查消除 → 更新分数 → 剩余步数-1
  ↓
判定结果：
  ├─ 分数 ≥ 目标分数 → 过关！显示星级评价
  └─ 步数用尽且分数不足 → 失败，可选重试或返回
```

### 3.3 关卡进度存档

- 记录玩家已通过的最高关卡
- 记录每关的最高分和星级
- **存储方式**：使用 Unity 的 `PlayerPrefs`（简单方案，够用即可）
