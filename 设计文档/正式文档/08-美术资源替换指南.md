# 08 - 美术资源替换指南

> 本文档说明游戏如何加载美术资源、当前的目录结构和映射关系，以及如何替换正式素材。  
> 面向技术美术和美术同学。

## 1. 资源加载机制

游戏通过 `Resources.Load<Sprite>()` 从 `Assets/Resources/Art/` 目录加载美术资源。  
如果资源不存在，会自动 fallback 到代码生成的纯色方块（不会崩溃）。

```csharp
// SpriteUtils.cs 中的加载逻辑
var sprite = Resources.Load<Sprite>(resourcePath);
return sprite != null ? sprite : fallbackSprite;
```

## 4. 替换方法

### 4.1 直接替换 PNG 文件（最简单）

1. 将正式 PNG 文件**替换** `Assets/Resources/Art/` 下的同名文件
2. 保持文件名不变
3. 回 Unity 等待自动刷新（或 `Ctrl+R`）
4. 运行游戏查看效果

### 4.2 添加 6 色独立方块（推荐，大幅提升视觉）

1. 将 6 个 PNG 放到 `Assets/Resources/Art/Blocks/`
2. 命名为 `blk_color1.png` ~ `blk_color6.png`
3. 代码中需修改 `SpriteUtils.cs` 和 `BlockSpawner.cs`（由 AI 完成）

### 4.3 替换音效文件

1. 将短音效放到 `Assets/Resources/Audio/SFX/`
2. 将背景音乐放到 `Assets/Resources/Audio/BGM/`
3. 如果只是替换同名文件，保持文件名不变即可
4. 如果新增音效，需要在 `AudioLibrary` / `AudioCue` 中重新拖拽引用

具体配置方式见 [操作指南/M3音效与反馈配置指南](../操作指南/M3音效与反馈配置指南.md)。

### 4.4 Unity Import Settings

| 设置项 | 值 |
|-------|-----|
| Texture Type | **Sprite (2D and UI)** |
| Sprite Mode | Single |
| Pixels Per Unit | **128** |
| Filter Mode | Bilinear |
| Compression | None 或 Low Quality |

## 5. 代码调用关系

```
SceneBootstrap.Awake()
├── SetupCamera()           → 配置相机
├── CreateBackground()      → 加载 bg_game.png 作为全屏背景
├── CreateManagers()
│   ├── BoardManager.Init()
│   │   └── CreateBoardVisuals()  → SpriteUtils.CellSprite (brd_cell.png)
│   └── BlockSpawner.Init()
│       └── CreateBlockVisual()   → SpriteUtils.BlockSprite (blk_base.png) + Color 着色
└── CreateUI()              → M3 改为 Prefab 后使用 UI 素材
```

## 6. 替换策略路线图

| 阶段 | 做什么 | 说明 |
|------|--------|------|
| M2（当前） | Resources.Load + fallback | 通过路径加载 Sprite，缺失时代码生成替代 |
| M3 | UI Prefab 化 | UI 改为 Prefab，在 Inspector 中拖拽赋值 Sprite |
| M4 | 全量替换 | 美术按 [05-美术资源规范](05-美术资源规范.md) 导出最终素材 |
