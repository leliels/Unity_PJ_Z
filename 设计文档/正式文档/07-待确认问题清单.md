## 🎮 玩法相关

- [x] **棋盘大小**：8×8 还是 10×10？还是可配置（不同关卡不同大小）？
  - 建议：先用 8×8 开发，后续扩展
  - 确认结果：**8×8，已实现。LevelData 中保留 boardRows/boardCols 字段供后续关卡配置不同大小。**

- [x] **游戏朝向**：横屏还是竖屏？
  - 建议：竖屏（1080×1920），适合手机单手操作
  - 确认结果：**竖屏 1080×1920，已在 SceneBootstrap 中配置 Camera 正交尺寸和 CanvasScaler。**

- [x] **方块是否可旋转**：玩家拖拽方块时能否旋转？
  - 参考：《快乐爱消除》**不支持**旋转
  - 建议：不支持旋转，降低复杂度
  - 确认结果：**不支持旋转，已确定。**

- [x] **候选方块数量**：每次显示几个候选方块？
  - 参考：《快乐爱消除》是 3 个
  - 建议：3 个
  - 确认结果：**3 个，已实现（Constants.CandidateCount = 3）。**

- [x] **候选方块刷新时机**：3 个都用完才刷新，还是用一个刷一个？
  - 参考：《快乐爱消除》是 3 个都用完才刷新
  - 建议：3 个都用完才刷新
  - 确认结果：**3 个都用完才刷新，已实现。**

- [x] **消除后方块是否下落**：消除一行后，上面的方块会不会掉下来？
  - 参考：《快乐爱消除》**不下落**
  - 建议：不下落（简化逻辑）
  - 确认结果：**不下落，已实现。**

- [x] **游戏结束条件**：
  - 关卡模式：步数用尽即结束
  - 是否还需要"无限模式"（无法放置任何方块时结束）？
  - 建议：MVP 只做关卡模式
  - 确认结果：**当前已实现的“无法放置即 GameOver”玩法定义为传统模式，M3 继续保留并从 Title 入口进入；关卡/步数玩法归入后续冒险模式设计。**

- [x] **道具系统**：MVP 阶段是否需要道具？
  - 建议：MVP 不做道具，后续再加
  - 确认结果：**MVP 不做道具。**

- [x] **方块形状列表**：需要确认最终的方块形状和出现概率
  - 初步方案见 [02-游戏设计文档](02-游戏设计文档.md) 的方块形状说明
  - 确认结果：**方块形状改为通过 `BlockShapeDatabase` 配置资产维护，并使用 `Tools / 方块形状配置器` 编辑。当前内置 18 种形状保留为 fallback。具体操作见 [操作指南/方块形状配置工具使用指南](操作指南/方块形状配置工具使用指南.md)。**

- [x] **方块方向**：方块是否需要多方向变体？
  - 参考：竞品不允许手动旋转，但每种方块会随机出现不同方向
  - 确认结果：**玩家不支持手动旋转；工具支持基于一个基础形状一键生成 0/90/180/270 四个独立方向变体，每个方向可独立启用和配置权重。**（2026-04-30 确认）

- [x] **消除视觉提示**：消除时是否有颜色覆盖效果？
  - 参考：竞品中消除前整排先变成手上方块的颜色
  - 确认结果：**需要实现。消除前整排先变色（覆盖为当前放置方块颜色）→ 短暂停留 → 再执行消除动画。**（2026-04-24 会议确认）

- [x] **商业模式**：游戏如何变现？
  - 确认结果：**广告变现为主。死亡后看广告可获得一条命（复活机会）。暂不做内购。后续可考虑方块皮肤系统。**（2026-04-24 会议确认）

## 🔧 技术相关

- [x] **是否引入 DOTween**：用于 UI 和方块动画效果
  - 优点：做动画非常方便，代码简洁
  - 缺点：多一个第三方依赖
  - 建议：引入，DOTween 非常成熟稳定
  - 确认结果：**暂不引入，MVP 阶段使用原生协程/动画，保持零外部依赖。后续体验不足时再考虑。**

- [x] **数据存储方案**：使用 PlayerPrefs 还是 JSON 文件？
  - 建议：MVP 用 PlayerPrefs（最简单），后续有需要再换
  - 确认结果：**PlayerPrefs，已确定。**

- [x] **目标平台优先级**：先做哪个平台？
  - 建议：先 PC（方便调试），后续打 Android/iOS
  - 确认结果：**先 PC（Standalone），已确定。**

- [x] **棋盘实现方案**：用代码动态生成 SpriteRenderer，还是用 Tilemap？
  - 建议：代码动态生成（更灵活，逻辑更清晰）
  - 确认结果：**代码动态生成 SpriteRenderer，已实现。**

- [x] **Title 设置范围**：Title 设置里放哪些功能？
  - 确认结果：**Title 设置只放音乐、音效、震动和清除用户数据；重新开始、返回、关闭等属于游戏内流程。**（2026-05-01 确认）

- [x] **音效配置方式**：音效是否写死在代码里？
  - 确认结果：**不写死在具体按钮代码里。采用 `AudioManager` 总控 + `AudioCue/AudioLibrary` 配置 + `UIAudioTrigger/ObjectAudioTrigger/BlockAudioFeedback/GameplayAudioBinder` 挂载或绑定。**（2026-05-01 确认）

- [x] **震动开发期表现**：PC 上如何验证震动？
  - 确认结果：**移动端调用系统震动；PC/Editor 下触发同一反馈接口，但表现为棋盘根节点或已放置方块容器短促抖动。**（2026-05-01 确认）
