# 06 - 开发计划

> 本文档是项目的整体开发路线图。  
> 我们按"里程碑"来推进，每个里程碑完成后都应该有一个可运行的版本。

## 1. 整体里程碑规划

```
M0 项目准备  →  M1 核心原型  →  M2 完整玩法  →  M3 内容填充  →  M4 打磨上线
```

| 里程碑 | 目标 | 预估周期 |
|--------|------|---------|
| **M0 - 项目准备** | 文档完善、美术风格确定、目录结构搭好 | 1 周 |
| **M1 - 核心原型** | 棋盘 + 方块放置 + 消除逻辑，能在编辑器里跑通 | 1~2 周 |
| **M2 - 完整玩法** | 关卡系统 + UI 流程 + 计分 + 存档，完整的一局游戏 | 2~3 周 |
| **M3 - 内容填充** | 制作 20~30 个关卡、接入美术素材和特效、音效 | 2~3 周 |
| **M4 - 打磨上线** | Bug 修复、体验优化、适配测试、准备上架 | 1~2 周 |

> 以上时间仅为估算，根据实际情况灵活调整。

## 2. 各里程碑详细任务

### M0 - 项目准备 ✅ 进行中

- [x] 创建 Unity 项目（2D URP 模板）
- [x] 初始化 Git 仓库
- [x] 搭建设计文档目录
- [ ] 确定美术风格方向（美术出概念图）
- [ ] 确定游戏是横屏还是竖屏
- [ ] 确定棋盘大小（8×8 / 10×10）
- [ ] 确认方块形状列表
- [ ] 搭建 Assets 目录结构

### M1 - 核心原型

**程序任务（AI 辅助）：**
- [ ] 创建 Singleton 基类
- [ ] 实现 BoardManager（创建棋盘、放置逻辑）
- [ ] 实现方块形状数据结构
- [ ] 实现 BlockSpawner（生成候选方块）
- [ ] 实现 BlockDrag（拖拽放置）
- [ ] 实现 MatchChecker（行列消除检测）
- [ ] 实现基础的 GameManager 状态管理

**美术任务：**
- [ ] 制作基础方块素材（6 色，临时素材即可）
- [ ] 制作棋盘格子素材
- [ ] 制作游戏背景

**验收标准：**
> 能在 Unity 编辑器中运行，看到棋盘，拖拽方块到棋盘上，填满一行后消除。

### M2 - 完整玩法

**程序任务：**
- [ ] 实现 ScoreManager（计分系统）
- [ ] 实现 LevelManager + LevelData ScriptableObject
- [ ] 实现 UIManager + 所有 UI 面板
- [ ] 实现关卡进度存档（PlayerPrefs）
- [ ] 实现 AudioManager
- [ ] 搭建 MainMenu 场景 + Game 场景
- [ ] 完成主菜单 → 关卡选择 → 游戏 → 结算的完整流程

**美术任务：**
- [ ] 制作 UI 素材（按钮、面板、图标）
- [ ] 制作游戏 Logo
- [ ] 制作主菜单背景

**验收标准：**
> 能从主菜单开始，选择关卡进入游戏，完成一关后看到结算，返回可以选下一关。

### M3 - 内容填充

**程序任务：**
- [ ] 制作 20~30 个关卡配置（ScriptableObject）
- [ ] 接入正式美术素材替换临时素材
- [ ] 接入特效
- [ ] 接入音效和背景音乐

**美术任务：**
- [ ] 制作正式的方块素材（最终风格）
- [ ] 制作所有 UI 最终稿
- [ ] 可能需要多套背景

**特效任务：**
- [ ] 制作消除特效
- [ ] 制作得分飘字
- [ ] 制作过关庆祝特效

**验收标准：**
> 游戏有完整的视觉和听觉体验，有足够的关卡内容可以玩上一阵子。

### M4 - 打磨上线

- [ ] 全面测试，修复 Bug
- [ ] 优化性能（如果需要）
- [ ] 调整关卡难度曲线
- [ ] 适配不同分辨率/设备
- [ ] 接入广告（如需要）
- [ ] 准备上架素材（图标、截图、描述）
- [ ] 打包发布

## 4. 风险与注意事项

| 风险 | 应对方式 |
|------|---------|
| 程序完全依赖 AI，出了 Bug 不好排查 | 每个功能做完立即测试，保持代码简洁 |
| 美术风格不统一 | 先确定风格再批量制作，做好参考图 |
| 关卡设计没经验 | 参考《快乐爱消除》等现有游戏的关卡设计 |
| 项目范围蔓延 | 严格控制 MVP 范围，"暂时不做"的坚决不做 |
