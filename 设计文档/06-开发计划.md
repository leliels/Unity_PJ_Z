> 本文档是项目的整体开发路线图。  
> 我们按"里程碑"来推进，每个里程碑完成后都应该有一个可运行的版本。

```
M0 项目准备  →  M1 核心原型  →  M2 数值体验+换皮  →  M3 完整玩法  →  M4 内容填充  →  M5 打磨上线
```

| 里程碑 | 目标 | 预估周期 | 状态 |
|--------|------|---------|------|
| **M0 - 项目准备** | 文档完善、美术风格确定、目录结构搭好 | 1 周 | ✅ 已完成 |

## 5. 关卡编辑工具方案

> 设计师如何编辑和创建关卡。

### 5.1 关卡编辑方式

采用 **ScriptableObject + Unity Inspector** 的方案：

1. **创建关卡**：在 Unity 中右键 → `Create` → `Game/LevelData`，创建一个关卡配置资产
2. **编辑参数**：在 Inspector 面板中直接填写关卡参数（目标分、步数、星级线等）
3. **存放位置**：统一放在 `Assets/ScriptableObjects/Levels/` 目录
4. **命名规范**：`Level_001.asset`、`Level_002.asset`...

### 5.2 关卡参数说明

| 字段 | 含义 | 建议值范围 |
|------|------|-----------|
| levelId | 关卡编号 | 1, 2, 3... |
| targetScore | 过关目标分 | 100~2000 |
| maxSteps | 可放置方块总次数 | 10~50 |
| boardRows / boardCols | 棋盘大小 | 默认 8×8 |
| star2Score | 2 星分数线 | 目标分 × 1.5 |
| star3Score | 3 星分数线 | 目标分 × 2.0 |

### 5.3 后续扩展（M3 阶段考虑）

- 预置方块（开局已有方块的关卡）
- 限制方块池（本关只出现特定形状）
- 自定义 EditorWindow 可视化关卡编辑器（带棋盘预览）

## 6. 美术资源替换流程

> 如何将临时程序生成的资源替换为正式美术素材。

### 6.1 当前状态

M1 阶段所有视觉元素都是**代码动态生成**的：
- 方块：`SpriteUtils.WhiteSquare`（纯色 32×32 纹理）+ 颜色着色
- 棋盘格子：同上，使用 `Constants.CellEmptyColor` 着色
- UI：代码创建 Canvas / Text / Button，无任何美术资源

### 6.2 替换策略（分两步走）

**第一步：M2 阶段 —— 改造为 Prefab + Sprite 引用**

1. 将代码动态创建的方块改为**加载 Sprite 资源**：
   - `Constants.cs` 中增加方块 Sprite 路径配置
   - `BlockSpawner.CreateBlockVisual()` 中从 `Resources` 或 Addressables 加载 Sprite
   - 同时保留纯色 fallback（美术未到位时不会报错）

2. 将棋盘格子改为使用美术提供的 Sprite：
   - `BoardManager.CreateBoardVisuals()` 中加载棋盘格子 Sprite

3. UI 改为 Prefab 方式（不再运行时代码创建）：
   - 制作 UI Prefab 放在 `Assets/Prefabs/UI/`
   - UIManager 通过 Instantiate 加载

**第二步：M3 阶段 —— 正式素材全量替换**

1. 美术按 [05-美术资源规范](05-美术资源规范.md) 的命名规范导出素材
2. 放到对应目录（`Art/Blocks/`、`Art/Board/`、`Art/UI/` 等）
3. 在 Unity 中设置 Texture Import Settings（Sprite 2D、Filter Mode 等）
4. 替换 Prefab 或配置中的 Sprite 引用

### 6.3 美术同学交付素材流程

```
美术制作素材 → 按命名规范导出 PNG → 放入 Art/ 对应子目录
→ 技术美术在 Unity 中配置 Import Settings → 替换引用 → 测试效果
```
