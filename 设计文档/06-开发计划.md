# 06 - 开发计划

> 本文档是项目的整体开发路线图。  
> 我们按"里程碑"来推进，每个里程碑完成后都应该有一个可运行的版本。

## 1. 整体里程碑规划

```
M0 项目准备  →  M1 核心原型  →  M2 数值体验+换皮  →  M3 完整玩法  →  M4 内容填充  →  M5 打磨上线
```

| 里程碑 | 目标 | 预估周期 | 状态 |
|--------|------|---------|------|
| **M0 - 项目准备** | 文档完善、美术风格确定、目录结构搭好 | 1 周 | ✅ 已完成 |
| **M1 - 核心原型** | 棋盘 + 方块放置 + 消除逻辑，能在编辑器里跑通 | 1~2 周 | ✅ 已完成 |
| **M2 - 数值体验 + 美术换皮** | 新计分系统（幂次消除分+Combo）+ 美术资源替换，可见人的版本 | 1~2 天 | 🔵 进行中 |
| **M3 - 完整玩法** | 关卡系统 + UI 流程 + 计分 + 存档，完整的一局游戏 | 2~3 周 | ⬜ 下一步 |
| **M4 - 内容填充** | 制作 20~30 个关卡、接入特效、音效 | 2~3 周 | ⬜ |
| **M5 - 打磨上线** | Bug 修复、体验优化、适配测试、准备上架 | 1~2 周 | ⬜ |

> 以上时间仅为估算，根据实际情况灵活调整。  
> **2026-04-23 调整**：根据主策划 Leliel 要求，在 M1 和原 M2 之间插入「M2 数值体验+换皮」，优先把产品做到"能见人"状态，快速推进。原 M2~M4 顺延为 M3~M5。

**验收结果：**

> 拖拽方块放置到棋盘，填满行/列自动消除并计分，无法放置时 GameOver，
> 点击 Restart 重新开始。核心玩法循环已跑通。

## 4. 风险与注意事项

| 风险 | 应对方式 |
|------|---------|
| 程序完全依赖 AI，出了 Bug 不好排查 | 每个功能做完立即测试，保持代码简洁 |
| 美术风格不统一 | 先确定风格再批量制作，做好参考图 |
| 两种模式差异化设计 | 明确冒险模式与传统模式的核心区别，避免同质化 |
| 项目范围蔓延 | 严格控制 MVP 范围，"暂时不做"的坚决不做 |
| 纯代码创建 UI 后续不好维护 | ✅ M2 已完成大量 Prefab 化，关键对象均可在 Inspector 中调整 |
