# M3 音效与反馈配置指南

> 用途：说明 M3 如何在 Unity Inspector 中配置音效和震动/抖动反馈。本文只写最常用流程。

## 1. 总体思路

音效分三层：

1. `AudioManager`：全局总控，管理音乐/音效开关、音量和播放。
2. `AudioCue` / `AudioLibrary`：配置“播什么声音、怎么播”。
3. 触发脚本：挂在按钮、方块或场景对象上，决定“什么时候播”。

不要把具体音效写死在按钮点击代码里。

## 2. AudioCue：单个音效

`AudioCue` 表示一个可播放音效。

| 参数 | 说明 |
|------|------|
| `Clips` | 可选音频列表；多个时可随机播放 |
| `Volume` | 本音效音量倍率，通常 `0~1` |
| `Delay` | 延迟播放时间，单位秒 |
| `Pitch Min/Max` | 播放速度/音高随机范围，默认 `1~1` |
| `Loop` | 是否循环；短音效一般关闭 |
| `Cooldown` | 冷却时间，防止短时间重复播放太多次 |
| `Use Global SFX Volume` | 是否受全局音效音量控制，通常开启 |

## 3. AudioLibrary：一套音效

`AudioLibrary` 用来集中管理一套音效配置。

常用字段建议：

| 字段 | 用途 |
|------|------|
| `UiClick` | 按钮点击 |
| `UiOpen` | 弹窗打开 |
| `UiClose` | 弹窗关闭 |
| `BlockPick` | 拾起方块 |
| `BlockDrop` | 放下方块 |
| `BlockPlace` | 成功放到棋盘 |
| `ClearLine` | 行/列消除 |
| `ScoreTick` | 跳分 |
| `GameOver` | 游戏结束 |

以后换一整套音效时，优先换 `AudioLibrary` 或其中的 `AudioCue` 引用。

## 4. UIAudioTrigger：按钮音效

挂载位置：按钮、弹窗、UI Prefab。

常用触发：

| 触发 | 用途 |
|------|------|
| `OnClick` | 点击按钮时播放 |
| `OnEnable` | 面板显示时播放 |
| `OnDisable` | 面板隐藏时播放 |

配置方法：

1. 选中按钮或面板 Prefab。
2. 添加 `UIAudioTrigger`。
3. 选择触发时机。
4. 拖入对应 `AudioCue`，如 `UiClick`。
5. 运行游戏点击测试。

## 5. ObjectAudioTrigger：对象音效

挂载位置：普通 GameObject、特效 Prefab、临时生成对象。

常用触发：

| 触发 | 用途 |
|------|------|
| `OnEnable` | 对象出现时播放 |
| `OnDisable` | 对象隐藏时播放 |
| `OnDestroy` | 对象销毁时播放 |
| `Manual` | 由动画事件或其他脚本手动触发 |

适合给消除特效、弹出物、提示物配置音效。

## 6. BlockAudioFeedback：方块音效

挂载位置：候选方块或拖拽方块 Prefab。

| 参数 | 用途 |
|------|------|
| `Pick Cue` | 拾起方块 |
| `Drag Cue` | 开始拖动 |
| `Drop Cue` | 松手放下 |
| `Place Success Cue` | 成功放置 |
| `Place Fail Cue` | 放置失败或取消 |

如果只想先验证链路，最少配置 `Pick Cue` 和 `Place Success Cue`。

## 7. GameplayAudioBinder：玩法事件音效

挂载位置：游戏场景中的管理对象，例如 `[Audio]` 或 `[GameplayAudioBinder]`。

它监听玩法事件，再播放音效：

| 事件 | 建议音效 |
|------|----------|
| 方块成功放置 | `BlockPlace` |
| 行/列消除 | `ClearLine` |
| Combo / 多行消除 | 可复用 `ClearLine` 或更强音效 |
| 跳分 | `ScoreTick` |
| GameOver | `GameOver` |

这种方式适合没有具体 UI 按钮或单个对象可挂的全局玩法事件。

## 8. FeedbackManager：震动与抖动

`FeedbackManager` 统一处理震动反馈。

| 平台 | 表现 |
|------|------|
| 移动端 | 调用系统震动 |
| PC / Editor | 棋盘根节点或已放置方块容器短促抖动 |

常用参数：

| 参数 | 说明 |
|------|------|
| `Enabled` | 是否启用震动/反馈 |
| `Duration` | 抖动时长，默认约 `0.5` 秒 |
| `Strength` | 抖动强度 |
| `Frequency` | 抖动频率 |
| `Damping` | 衰减，越高越快停止 |
| `Target Root` | PC 下被抖动的根节点，建议用棋盘或方块容器 |

## 9. 最小验证流程

1. 准备一个 placeholder 音效，如 `snd_ui_click.wav`。
2. 创建 `AudioCue`，把该音效拖到 `Clips`。
3. 创建 `AudioLibrary`，把 `AudioCue` 填到 `UiClick`。
4. 场景中放置 `AudioManager`，指定 `AudioLibrary`。
5. 给按钮加 `UIAudioTrigger`，选择 `OnClick`，引用 `UiClick`。
6. 运行游戏，点击按钮，确认能听到声音。

## 10. 常见问题

### 没声音

检查：`AudioManager` 是否存在、全局音效是否开启、`AudioCue` 是否有 Clip、音量是否为 0。

### 声音太吵

降低 `AudioCue.Volume`，或设置 `Cooldown` 防止重复播放。

### 想换音效

优先替换 `AudioCue` 里的 Clip；如果文件同名替换，保持路径和文件名不变即可。
