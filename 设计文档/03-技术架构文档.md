# 03 - 技术架构文档

> 本文档是项目的技术核心文档，主要面向程序开发（含 AI 辅助编码）。  
> 设计原则：**简洁、够用、易维护**。不引入复杂框架，保持原生 Unity 开发方式。

## 1. 架构设计原则

1. **Manager 单例模式**：核心系统用少量 Manager 管理，每个 Manager 职责清晰
2. **数据驱动**：关卡配置与代码逻辑分离，用 ScriptableObject 存储关卡数据
3. **简单状态机**：用枚举管理游戏状态，不引入复杂状态机框架
4. **不过度设计**：够用就好，不为"可能用到"的功能预留复杂接口

## 4. 核心 Manager 系统

### 4.1 系统总览

```
GameManager（游戏总管理）
├── BoardManager    - 棋盘逻辑（放置、消除）
├── LevelManager    - 关卡加载、目标判定
├── ScoreManager    - 计分
├── UIManager       - 界面切换与更新
├── AudioManager    - 音效播放
└── BlockSpawner    - 候选方块生成
```

### 4.2 各 Manager 职责

#### GameManager（游戏总管理器）

- 管理游戏全局状态（主菜单 / 游戏中 / 暂停 / 结算）
- 协调各 Manager 之间的通信
- 控制游戏流程（开始、暂停、结束）
- **生命周期**：全局单例，不销毁

#### BoardManager（棋盘管理器）

- 创建和维护棋盘数据（二维数组）
- 处理方块放置逻辑（判断是否可放、执行放置）
- 调用 MatchChecker 检测并执行消除
- **生命周期**：游戏场景中存在

#### LevelManager（关卡管理器）

- 加载关卡配置数据（ScriptableObject）
- 管理当前关卡的步数和目标
- 判定关卡是否通过
- 管理关卡进度存档
- **生命周期**：全局单例

#### ScoreManager（计分管理器）

- 计算放置得分、消除得分
- 管理连消奖励
- 提供当前分数查询
- **生命周期**：游戏场景中存在

#### UIManager（UI 管理器）

- 管理所有 UI 面板的显示/隐藏
- 更新游戏中 HUD（分数、步数等）
- **生命周期**：全局单例

#### AudioManager（音效管理器）

- 播放音效（SFX）和背景音乐（BGM）
- 管理音量设置
- **生命周期**：全局单例，不销毁

### 4.3 Manager 之间的通信方式

采用**直接引用 + C# 事件**的简单方式：

```csharp
// 示例：方块放置后的事件链
// BoardManager 放置方块 → 触发事件
public static event Action<int> OnBlockPlaced;      // 参数：放置的格子数
public static event Action<int> OnLinesCleared;     // 参数：消除的行/列数

// ScoreManager 监听事件 → 计算分数
// UIManager 监听事件 → 更新显示
// AudioManager 监听事件 → 播放音效
// LevelManager 监听事件 → 判定是否过关
```

> 不使用消息总线、不使用复杂的事件系统，直接用 C# 原生的 `event` / `Action` 就够了。

```csharp
// 棋盘用一个二维数组表示
bool[,] boardGrid;  // true = 已占用, false = 空

int rows = 8;
```

### 5.2 方块形状数据

```csharp
// 方块形状用相对坐标数组表示
// 例如 L 形：
// ■

[System.Serializable]
public class BlockShapeData
```

## 6. 游戏状态管理

```csharp
public enum GameState
{
    MainMenu,       // 主菜单
    LevelSelect,    // 关卡选择
    Playing,        // 游戏进行中
    Paused,         // 暂停
    GameOver,       // 游戏结束（失败）
    LevelComplete   // 关卡完成（过关）
}
```

状态切换由 `GameManager` 统一管理，切换时通知 `UIManager` 更新界面。

## 8. 代码规范

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | `BoardManager` |
| 公有方法 | PascalCase | `PlaceBlock()` |
| 私有方法 | PascalCase | `CheckMatch()` |
| 公有变量 | camelCase | `currentScore` |
| 私有变量 | _camelCase | `_boardGrid` |
| 常量 | UPPER_SNAKE | `MAX_BOARD_SIZE` |
| 枚举值 | PascalCase | `GameState.Playing` |

### 其他约定

- 每个脚本文件只放一个类
- `MonoBehaviour` 脚本用 `[SerializeField]` 暴露私有字段，避免用 `public`
- 优先用 `[SerializeField]` + Inspector 赋值，减少 `Find` / `GetComponent` 调用
- 注释用中文，方便团队阅读

## 10. 关卡编辑工具

### 10.1 编辑方式

采用 **ScriptableObject + Unity Inspector** 方案，设计师直接在 Inspector 面板中配置关卡参数：

```
右键 Assets 面板 → Create → Game → LevelData → 在 Inspector 中编辑参数
```

### 10.2 LevelData 字段说明

```csharp
[CreateAssetMenu(fileName = "Level_001", menuName = "Game/LevelData")]
public class LevelData : ScriptableObject
{
    public int levelId;         // 关卡编号（用于排序和存档）
    public int targetScore;     // 过关目标分数
    public int maxSteps;        // 最大放置步数
    public int boardRows = 8;   // 棋盘行数（默认 8）
    public int boardCols = 8;   // 棋盘列数（默认 8）
    public int star2Score;      // 2 星分数线
    public int star3Score;      // 3 星分数线
}
```

### 10.3 关卡文件组织

```
Assets/ScriptableObjects/Levels/
├── Level_001.asset     # 第 1 关
├── Level_002.asset     # 第 2 关
├── Level_003.asset     # 第 3 关
└── ...
```

### 10.4 关卡设计建议

| 难度 | 步数 | 目标分 | 说明 |
|------|------|--------|------|
| 简单 | 20~30 | 100~300 | 前 5 关，让玩家熟悉操作 |
| 中等 | 15~25 | 300~600 | 需要有意识地制造消除 |
| 困难 | 10~20 | 500~1000 | 需要多行同时消除才能达标 |

### 10.5 后续扩展（M3）

- 预置方块：在 LevelData 中加 `Vector2Int[] preplacedBlocks` 字段
- 限制方块池：`BlockData[] allowedShapes` 控制本关出现哪些形状
- 可视化编辑器：自定义 EditorWindow，可以在棋盘网格上点击放置预设方块

## 11. 美术资源接入流程

### 11.1 M1 阶段（当前）

所有视觉元素由代码运行时生成：
- `SpriteUtils.WhiteSquare`：32×32 白色纹理 + 颜色着色
- `Constants.BlockColors[]`：6 种方块颜色
- `SceneBootstrap.CreateUI()`：代码创建 Canvas/Text/Button

### 11.2 M2 阶段 —— 资源加载框架

改造为**可替换**架构，核心思路：

```csharp
// Constants.cs 中增加资源路径配置
public static class ArtConfig
{
    // 方块 Sprite 路径（如果存在则使用，否则 fallback 到纯色）
    public static readonly string[] BlockSpritePaths = {
        "Art/Blocks/blk_red",
        "Art/Blocks/blk_orange",
        // ...
    };
    
    // 棋盘格子 Sprite
    public static readonly string CellSpritePath = "Art/Board/brd_cell";
    
    // 背景
    public static readonly string GameBgPath = "Art/Backgrounds/bg_game";
}
```

替换时只需：
1. 把美术素材放到对应路径
2. 确保 Texture Import Settings 正确（Sprite 2D、Pixels Per Unit 等）
3. 代码会自动加载使用

### 11.3 M3 阶段 —— 全量替换

1. 美术按 [05-美术资源规范](05-美术资源规范.md) 导出最终素材
2. 统一放入 `Assets/Art/` 对应子目录
3. UI 改为 Prefab 方式，在 Inspector 中拖拽赋值 Sprite
4. 测试所有界面效果
